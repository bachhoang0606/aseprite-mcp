# Subject reference conventions

In-repo, linkable reference notes for recurring subjects. These capture the
agreed conventions so every redraw stays consistent (instead of re-deciding the
design each session). Pair with `rules/03-proportions-silhouette-3-4-view.md`.

- [`goblin.md`](goblin.md) — the project's goblin: proportions, palette, rig, views.
- [`pixel-art-sources.md`](pixel-art-sources.md) — cited real-world pixel-art
  references (Aseprite docs, Derek Yu, Saint11, Lospec, Pixel Parmesan…) and the
  synthesized techniques behind `rules/` and `skills/`.

Add a new file per recurring subject. Keep each: silhouette intent, proportions,
palette (link to `knowledge/palettes/`), rig (link to `rules/05`), per-view notes.
